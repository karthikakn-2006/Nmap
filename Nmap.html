NETWORK RECONNAISSANCE

NMAP

Discover live hosts, map open ports, fingerprint services and operating systems — the first move in any security assessment.
Scan Types
RECON

-sn Ping Scan

Finds live hosts on a network without touching any ports.

$ sudo nmap -sn 192.168.1.0/24
Host is up (0.002s latency)
DEFAULT

nmap Basic Scan

Scans the top 1,000 most common TCP ports on a target.

$ nmap 192.168.1.5
22/tcp open ssh
80/tcp open http
DEEP

-sV Version Detection

Identifies exact service names and version numbers behind open ports.

$ sudo nmap -sV 192.168.1.5
22/tcp OpenSSH 8.2p1
FINGERPRINT

-O OS Detection

Analyzes packet responses to guess the target's operating system.

# Running: Linux 5.X
AUTOMATE

--script NSE Engine

Runs scripts for automated vulnerability detection and recon.

$ nmap --script vuln 192.168.1.5
REPORT

-oN Save Output

Saves scan results to a file for reporting and later analysis.

$ nmap 192.168.1.5 -oN results.txt
Port States

🟢 Open

A service is actively listening and accepting connections on this port.

🔴 Closed

The host is reachable but no service is listening on this port.

🟡 Filtered

A firewall or filter is blocking Nmap from determining the true state.

⚡ Key Takeaway

  • Start every assessment with host discovery (-sn) before scanning ports
  • Combine -sV and -O to build a full picture of a target's stack
  • NSE scripts turn Nmap from a scanner into a lightweight vuln checker
  • Always save output (-oN/-oX) — undocumented recon is wasted recon
  • Only scan networks and hosts you own or are explicitly authorized to test

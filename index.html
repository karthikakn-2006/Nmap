Nmap Step-by-Step

Discover · Enumerate · Report
DAY 05

1 Install Nmap

Kali: pre-installed. Ubuntu/Debian: install via apt. Windows/macOS: use official installer from nmap.org.

$ sudo apt install nmap
# or download installer: nmap.org/download.html

2 Find Live Hosts

Ping scan — disables port scanning, just checks which devices are online. Run before heavy scans.

$ sudo nmap -sn 192.168.1.0/24
# replace with your own subnet
Nmap scan report for 192.168.1.5
Host is up (0.002s latency)

3 Basic Port Scan

Default scan checks the top 1,000 most common TCP ports on a target.

$ nmap 192.168.1.5
PORT    STATE  SERVICE
22/tcp  open   ssh
80/tcp  open   http

4 Service & Version Scan

The -sV flag identifies exactly which service and version is running on each open port.

$ sudo nmap -sV 192.168.1.5
22/tcp  open  ssh    OpenSSH 8.2p1
80/tcp  open  http   Apache 2.4.41

5 OS Detection

The -O flag sends packets and analyzes responses to guess the target's operating system.

$ sudo nmap -O 192.168.1.5
Running: Linux 5.X
OS details: Linux 5.4 - 5.15

6 NSE Scripts

Nmap Scripting Engine automates vulnerability checks and admin tasks.

$ nmap --script vuln 192.168.1.5
# runs default vulnerability detection scripts

7 Save Scan Results

Use -oN to save output as a plain text file for reporting and later analysis.

$ nmap 192.168.1.5 -oN scan_results.txt
# -oX for XML output, -oG for grepable format

Key Takeaway

  • Always start with a ping scan (-sn) to find live hosts first
  • -sV gives service/version, -O guesses the OS
  • NSE scripts (--script vuln) automate vulnerability checks
  • Save every scan (-oN/-oX) for reporting later

Next Steps

  • Try Zenmap for a GUI-based workflow
  • Only scan networks/hosts you own or are authorized to test
